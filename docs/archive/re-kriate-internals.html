The ctx Object state

Context — single source of truth

All application state lives in one ctx table, created in app.init() and threaded through every function call. No globals, no module-level state.

ctx = { tracks = { track1, track2, track3, track4 }, -- 4 tracks active_track = 1, -- currently selected (1-4) active_page = "trigger", -- trigger|note|octave|duration|velocity playing = false, -- sequencer running? loop_held = false, -- grid key 12 held? loop_first_press = nil, -- first column of loop edit grid_dirty = true, -- needs grid redraw? scale_notes = { ... }, -- MIDI notes from musicutil g = grid.connect(), -- grid device handle grid_metro = metro, -- 30fps redraw timer clock_ids = { ... }, -- per-track clock coroutine IDs }

Track Data Model lib/track.lua

Track Structure
track = { division = 1, -- clock division index (1=1/16 .. 7=1/1) muted = false, params = { trigger = { steps={1,0,1,0,...}, loop_start=1, loop_end=16, pos=1 }, note = { steps={1,2,3,4,...}, loop_start=1, loop_end=8, pos=1 }, octave = { steps={4,4,4,4,...}, loop_start=1, loop_end=16, pos=1 }, duration = { steps={3,3,3,3,...}, loop_start=1, loop_end=16, pos=1 }, velocity = { steps={5,4,5,4,...}, loop_start=1, loop_end=16, pos=1 }, } }

Each param has 16 steps, independent loop start/end, and its own playhead position. This is the core of kria's polymetric design — different loop lengths per parameter create phase-shifting patterns.

Step Value Ranges
trigger: 0 or 1
note: 1-7 (scale degree)
octave: 1-7 (4 = center)
duration: 1-7 (mapped to beats)
velocity: 1-7 (mapped to 0.15–1.0)
Key Functions
advance(param) — step forward, return value, wrap at loop_end
toggle_step(param, x) — flip trigger 0/1
set_step(param, x, val) — set value page step
set_loop(param, s, e) — change loop bounds, clamp pos

Polymetric Loops core concept

Independent loop lengths per parameter

Each parameter advances its own playhead independently. With different loop lengths, the combination of trigger + note + octave produces evolving patterns that only repeat at the LCM of all loop lengths.

Sequencer Pipeline lib/sequencer.lua

Seamstress gap. These 8 norns APIs have no seamstress equivalents yet. The data model (lib/track.lua) and grid UI logic (lib/grid_ui.lua) are platform-agnostic — only app.lua and sequencer.lua need a compatibility layer for clock, params, metro, nb, and musicutil.
