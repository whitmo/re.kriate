polymetric sequencing

re.kriate tutorial — per-parameter loop lengths & evolving patterns

In most sequencers, all parameters share the same loop length. Step 1 of the trigger pattern always plays with step 1 of the note pattern. The result is a fixed phrase that repeats identically.

re.kriate does something different. Each parameter — trigger, note, octave, duration, velocity — has its own independent loop length. They cycle at different rates, creating combinations that don't repeat for a very long time.

This is polymetric sequencing.

1. per-parameter loops

Each parameter has its own loop_start and loop_end. When a parameter's playhead reaches loop_end, it wraps back to loop_start. Meanwhile, other parameters keep advancing at their own pace.

Imagine five runners on circular tracks of different sizes. They all start at the same moment, but each returns to the start line at a different time. The combination of their positions is always shifting.

2. the math: least common multiple

When do all parameters realign? At the LCM (least common multiple) of their loop lengths. With loops of 3 and 4, you get 12 unique steps before the pattern repeats. With 5 different prime-ish lengths, the LCM can be enormous.

3. four tracks, infinite variation

re.kriate runs 4 tracks simultaneously, each with its own clock division and 5 independent parameter loops. That's up to 20 independent cycles generating an ever-shifting musical texture.

Each track also has its own direction mode: forward, reverse, pendulum, drunk (random walk), or random. Combined with per-parameter loops, even a few programmed steps produce vast generative sequences.

5. putting it together

trigger (loop 1–16) — when a note fires (binary on/off)

note (loop 1–16) — scale degree (1–7)

octave (loop 1–16) — octave offset (1–7, center = 4)

duration (loop 1–16) — note length (1/16 to 4 beats)

velocity (loop 1–16) — amplitude (0.15 to 1.0)

By programming short melodic fragments into note and octave, rhythmic patterns into trigger, and varying duration and velocity with different loop lengths, you create sequences that breathe and evolve — always recognizable, never quite the same.

On the grid, hold the loop modifier button (row 8, col 12) and press two steps to set loop start/end for the active parameter. Each parameter page shows its own loop region independently.
