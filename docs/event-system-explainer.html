re.kriate Event System

A lightweight pub/sub event bus for decoupled communication between sequencer, voice, grid, and UI modules.

Architecture

The event bus sits at the center of re.kriate, allowing modules to communicate without direct references to each other. Publishers emit events; subscribers react to them. Neither needs to know the other exists.

Event Flow Example

When the sequencer advances a step, here is what happens:

1. Sequencer
advance step
→
2. Event Bus
emit("sequencer:step")
→
3a. Voice
play note
→
3b. Grid UI
update LEDs
→
3c. Screen
redraw display

All three subscribers react independently. The sequencer does not need to know about voices, grids, or screens.

Wildcard Matching

Subscribe with "prefix:*" to match all events in a namespace. The wildcard matches any suffix after the first colon.

sequencer:*
sequencer:start sequencer:stop sequencer:step sequencer voice:note
voice:*
voice:note voice:all_off voice grid:key
grid:*
grid:key grid:redraw grid track:select

Wildcard handler signature

Wildcard handlers receive the full event name as the first argument, so you can distinguish which specific event fired:

-- Exact match: handler(data)
bus:on("voice:note", function(data)
  print(data.note)  -- data is the first arg
end)

-- Wildcard: handler(event_name, data)
bus:on("voice:*", function(event_name, data)
  print(event_name)  -- "voice:note" or "voice:all_off"
  print(data.note)
end)

Common Patterns

Basic usage

local events = require("lib/events")
local bus = events.new()

-- Subscribe
local unsub = bus:on("voice:note", function(data)
  print("Note:", data.note, "Track:", data.track)
end)

-- Emit
bus:emit("voice:note", {track = 1, note = 60, vel = 0.8})

-- Unsubscribe
unsub()

Integration with context object

-- In app.init(), create the bus on the context
function M.init()
  local ctx = {}
  ctx.events = events.new()

  -- Wire up voice to respond to sequencer steps
  ctx.events:on("sequencer:step", function(data)
    voice.play(ctx, data.track, data.note, data.vel, data.dur)
  end)

  return ctx
end

One-shot events

-- React to the first start only
bus:once("sequencer:start", function(data)
  print("Sequencer started for the first time!")
end)

Logging / debugging

-- Log every event in a namespace
bus:on("sequencer:*", function(event_name, data)
  print("[debug]", event_name, inspect(data))
end)

Decoupled Testing

The event bus makes modules independently testable. Instead of mocking complex module interfaces, subscribe to events and assert on the data.

✓

Assert on emitted events

Subscribe in tests, collect events, assert that the right events were emitted with the right data.

✓

No module mocking needed

Modules communicate through the bus. To test the sequencer, you do not need to mock the voice -- just check the events.

✓

Replay and simulate

Emit events programmatically to simulate grid presses, sequencer steps, or param changes without hardware.

-- Test example: verify sequencer emits step events
local bus = events.new()
local received = {}
bus:on("sequencer:step", function(data)
  table.insert(received, data)
end)

sequencer.advance(ctx)  -- triggers bus:emit internally

assert.are.equal(1, #received)
assert.are.equal(1, received[1].track)
