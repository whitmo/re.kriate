How Testing Works

Framework & Tools

1
Busted — Lua test framework with BDD-style syntax

describe, it, before_each, after_each organize test structure

2
Luassert — Assertion library bundled with Busted

assert.are.equal, assert.are.same, assert.is_true, assert.is_not_nil

3
LuaCov — Code coverage analysis

Configured via .luacov to track only lib/ files, excluding test dependencies

4
GitHub Actions CI — Runs on every push and PR

Lua 5.4 install → lint (luac -p) → test with coverage → threshold check (80%)

Test Architecture: Mock Everything External

Tests run on standard Lua without norns/seamstress hardware. All platform APIs are mocked.

M
Mock clock — get_beats(), run(), cancel(), sync()

Controls time. In ratchet tests, clock.run executes immediately for synchronous testing.

M
Mock params — add_number, add_option, get, set, set_action

Simulates the norns parameter system. Actions fire on set() just like real norns.

M
Mock grid — led(x, y, brightness), refresh(), all()

Records LED calls. Tests verify brightness values at specific grid coordinates.

M
Mock musicutil — generate_scale(root, type, octaves)

Returns a deterministic whole-tone-like scale so tests can predict exact MIDI note values.

V
Recorder Voice — The test-dedicated voice backend

Captures play_note() calls into an event buffer. Tests assert on captured note, velocity, duration, and track number. Shared buffer enables multi-track assertions.

Running Tests

# Run all tests with coverage
busted --coverage specs/

# Generate coverage report
luacov

# View coverage summary
tail -20 luacov.report.out

# Run a single spec file
busted specs/sequencer_spec.lua

# Run tests matching a pattern
busted --filter "ratchet" specs/

Test Patterns Used

Context Object Pattern
  • All state flows through a single ctx table
  • Each test creates a fresh ctx via make_ctx()
  • No global state pollution between tests
  • Modules receive and operate on context, never globals
Property-Based Testing
  • Direction modes tested with 200+ iterations
  • Verifies positions stay within loop bounds
  • Catches edge cases in random/drunk walk
  • Ensures wrapping correctness under stress
Shared Buffer Assertions
  • Recorder voice captures all events to a buffer
  • Multiple voices share one buffer
  • Tests filter by track number to verify routing
  • Enables end-to-end integration assertions
Grid Coordinate Verification
  • Mock grid records LED brightness per (x,y)
  • Tests verify exact brightness at specific coordinates
  • Validates bar graph rendering, playhead highlight
  • Checks nav button states (dim=3, bright=12, head=15)
