## Summary

Complete the norns entrypoint (`re_kriate.lua`) to full feature parity with the seamstress entrypoint. The current 55-line stub has nb voice setup and basic callback delegation but lacks explicit grid provider config, logging lifecycle, screen metro refresh, and nb_voice portamento. All changes are confined to two existing files: `re_kriate.lua` and `lib/norns/nb_voice.lua`. No new modules needed.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

### I. Context-Centric Architecture — PASS
All state flows through `ctx` created by app.init(). The entrypoint defines only the 5 norns global hooks which delegate to app module functions. nb_voice instances are injected via config.voices. Screen metro stored on ctx. No custom globals, no hidden module state.

### II. Platform-Parity Behavior — PASS
Sequencing behavior is identical across platforms — all shared logic lives in app.lua, sequencer.lua, track.lua, etc. The norns entrypoint differs only in platform adapters: nb voices (vs MIDI/OSC), monome grid (vs simulated), and norns screen API (vs seamstress screen). These are expected platform-specific differences. nb_voice.set_portamento adds norns parity for glide (seamstress already has MIDI CC portamento and OSC portamento).

### III. Test-First Sequencing Correctness — PASS
nb_voice.set_portamento affects voice output (portamento behavior). Tests will be written before implementation per TDD ordering. Entrypoint wiring tests verify delegation correctness. No changes to sequencing logic itself (sequencer.lua, track.lua unchanged).

### IV. Deterministic Timing and Safe Degradation — PASS
No new clock/timing code. Screen metro uses standard metro.init() at 15fps (identical pattern to existing grid_metro and seamstress screen_metro). nb_voice.set_portamento gracefully degrades when player doesn't support set_slew (no error, no crash).

### V. Spec-Driven Delivery — PASS
Full speckit pipeline: spec.md → plan.md → tasks.md → analyze → implement → verify. All artifacts in specs/005-norns-entrypoint/. Feature queued in feature-queue.md.

## Implementation Phases

### Phase 1: nb_voice Portamento (FR-006, US3)

Add set_portamento(time) to lib/norns/nb_voice.lua that calls player:set_slew(time) if the player supports it, otherwise no-ops gracefully.

**Test tasks**: Write tests for set_portamento with supporting player, non-supporting player, and nil player.
**Implementation**: Add set_portamento method to the nb_voice.new() return table.

**Files**: lib/norns/nb_voice.lua, specs/norns_entrypoint_spec.lua
